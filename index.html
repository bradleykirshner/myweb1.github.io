HTTP is a protocol that is used to transmit documents over the internet. The current version is HTTP/1.1 but there are newer versions such as HTTP/2 and HTTP/3. HTTP has request methods to indicate the action wanted to be performed on the server. Some common HTTP methods consist of GET, POST, PUT, DELETE, etc. HTTP communication involves request made from clients to servers and responses from servers to clients. HTTP headers provide metadata about the HTTP message. Some common header fields consist of Content-Type, Content-Length, Cache-Control, User-Agent, etc. Response status codes indicate the outcome of a clients request to the server. Some common response status codes consist of 100 codes, 200 codes, 300 codes, 400 codes, and 500 codes.

RESTful services enable clients to perform CRUD operations on server resources using standard HTTP methods. REST stands for representational state transfer. RESTful API endpoints are URLs that represent resources on the server. They define the entry points for interacting with those resources. Routing is the process of determining how an incoming HTTP request should be handled and which code should be executed to generate the appropriate response. Routing lets developers define the structure and behavior of web applications by directing request to the appropriate resources. Clean URLs are user friendly URLs. They are easy for humans to read. Clean URLs improve usability for users by giving them clear and descriptive links. Resources are an object or data entity that can be accessed and sometimes modifiable by clients through the API. Endpoints are specific URLs in a RESTful API that correspond to a particular resource or collection of resources.

The POST method is used to submit data to the server. The GET method is used to request data from a server. The PUT method is used to update data or submit new data if it does not exist. The PATCH method is used to apply partial modifications to a resource. The DELETE method is used to delete a specified resource. The OPTIONS method is used to describe the communication options for the specified resource. The HEAD method is used to ask for a GET request, but without the response body. The TRACE method is used to perform a message loop-back test along the path to the specified resource. The CONNECT method is used to establish a tunned to the server identified by the specified resource.
